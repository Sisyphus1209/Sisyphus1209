### 1. AI初创公司Rocket以远低于麦肯锡的价格提供风格类似的氛围报告。
*TechCrunch* | 05:30

印度初创公司Rocket正押注下一个重大机遇将出现在"氛围编程"之前的环节：让人工智能帮助人们决定开发什么产品。该公司已推出一款能生成咨询式产品策略的平台。

这家总部位于印度苏拉特的初创公司于周二发布了Rocket 1.0平台，该平台将市场研究、产品构建和竞争情报整合到统一工作流中。该平台可生成详细的产品战略文档，涵盖定价策略、单位经济效益及市场进入建议等核心内容。

随着Cursor、Replit、Lovable等AI编程平台以及Claude Code、Codex等功能的涌现，代码编写已变得前所未有的便捷高效。"现在每个人都能生成代码……这已成为标准化能力。但究竟该开发什么产品，却是众人皆感迷茫的领域，"Rocket联合创始人兼首席执行官维沙尔·维拉尼（上图）指出，并强调"运营企业与单纯构建代码库是两回事"。

TechCrunch在平台发布前进行了简短测试，发现其能根据简单指令生成PDF格式的产品需求文档。这些文档更接近专业咨询报告，而非侧重功能实现的氛围编程工具或聊天机器人。

不过部分分析内容似乎是对现有数据的整合——融合了已知定价模型、用户行为模式和竞争洞察——而非基于可独立验证的信息。这意味着用户在进行商业决策前仍需验证输出结果。维拉尼表示，当用户遇到问题时，平台可提供人工支持服务。

该产品还能追踪竞争对手动态，包括其网站改版和流量趋势变化。维拉尼透露，Rocket的分析系统整合了超1000个数据源，涵盖Meta广告库、Similarweb应用程序接口及自有网络爬虫。

Rocket的订阅方案分为三档：应用构建版每月25美元，战略研究版每月250美元，包含竞争情报的完整平台版最高350美元。维拉尼向TechCrunch透露，250美元档方案在构建产品的同时，还能生成2-3份"麦肯锡级别"的研究报告，其高端服务以远低于传统咨询（同类战略工作通常需数万美元）的成本提供替代方案。

去年9月，Rocket获得由Accel、Salesforce Ventures和Together Fund投资的1500万美元种子轮融资。该公司宣称其用户量已从40万增长至覆盖180个国家的超150万，年度用户平均收入约4000美元（但未披露具体付费客户数）。这家毛利率超50%的初创公司表示，其20-30%的客户为中小企业。

目前Rocket拥有57名员工，总部设于苏拉特，并在帕洛阿尔托设有运营中心。

### 2. AI设计平台Picsart推出创作者变现计划
*TechCrunch* | 04:01

AI设计平台Picsart近日向TechCrunch独家透露，公司将推出创作者变现计划。该项目向所有创作者开放，无需邀请名单，不设粉丝量门槛。

该计划邀请创作者使用Picsart工具为特定活动创作原创内容，在其社交渠道发布，并根据受众互动情况获得收益。Picsart表示，该项目旨在奖励创意产出和内容表现，而非单纯追求规模或粉丝数量。

通过推出创作者变现计划，Picsart正从创意工具转型为创作者创收平台。公司可能将此视为吸引和留住创作者的双赢举措。

创作者注册后即可访问控制面板，查看所有可参与的创作提示和创意挑战。例如，某次活动可能要求创作者使用Picsart Aura（该平台的AI对话助手，可通过文字或语音提示生成并动态化图像视频）制作可爱的毛绒生物。

Picsart特别指出，缺乏创意投入的简单AI图像生成与发布无法带来实质性互动或收益。

内容创作完成后，创作者需填写简短表格提交社交媒体上的作品链接，根据活动要求添加标签，并简要说明在Picsart中的创作过程。内容将直接发布至创作者个人的Instagram、TikTok、YouTube和X账号。

收益根据浏览量、评论量、分享量和触达量计算。创作者可通过控制面板实时追踪收益，并通过Stripe提现。

"我们始终相信创意应属于每个人——如今，收益也应如此，"Picsart创始人兼首席执行官霍夫汉尼斯·阿沃扬在声明中表示，"创作者经济存在结构性难题：平台从未真正致力于补偿普通创作者。Picsart变现计划是我们对数百万社区建设者的承诺。它开放、有序且直接——参与创作，只要内容表现优异，就能获得报酬。"

公司表示，无论是制作教程、美学编辑还是短视频，各类创作者都能获得收益机会。

成立于2011年的Picsart在全球拥有超1.3亿用户，在2021年创作者经济热潮中跻身独角兽行列。此次拓展创作者变现业务，距离其宣布推出AI智能体市场仅数周——该市场允许创作者为不同任务"雇佣"AI助手，例如调整社交媒体内容尺寸与混剪，或编辑Shopify产品照片。

### 3. OpenAI前员工正悄然运作一只新基金，规模或达1亿美元。
*TechCrunch* | 21:54

一家与OpenAI渊源深厚的新风险投资基金已初步完成其1亿美元募资目标的首次交割，创始人向TechCrunch透露。该基金合伙人已开始进行首批投资。

这家名为"零样本"（Zero Shot，源自AI训练术语的创意变体）的基金，其联合创始团队包含数位因缘际会转型为风投人的OpenAI元老。三位创始合伙人来自OpenAI：埃文·森川（Evan Morikawa）曾领导DALL·E、ChatGPT至Codex发布期间的应用工程团队，现任职机器人初创公司Generalist；安德鲁·梅恩（Andrew Mayne）作为OpenAI首位提示工程师，以主持OpenAI播客闻名，他还创立了AI部署咨询公司Interdimensional；肖恩·贾恩（Shawn Jain）则是工程师出身的OpenAI前研究员，后转型风投并创立生成式AI初创公司Synthefy。

团队还吸纳了风投人凯利·科瓦奇（Kelly Kovacs），她曾是迪克·科斯特洛（Dick Costello）与亚当·贝恩（Adam Bain）创立的成长期风投机构01A的创始合伙人。第五位创始成员布雷特·朗萨维尔（Brett Rounsaville）来自推特和迪士尼，同时担任梅恩旗下Interdimensional的CEO。

梅恩告诉TechCrunch，这些OpenAI校友"相识多年"，从ChatGPT发布前到公司爆发式增长期始终在模型制造商并肩作战。离职后，他们频繁收到风投机构关于新兴AI技术的咨询邀约，以及创始人朋友的求教，这促使梅恩创立了咨询公司。"我们有些朋友离开OpenAI后想创业。"梅恩表示。

他们发现众多获得融资的AI初创公司与市场需求存在巨大鸿沟。"或许该成立自己的基金，因为我们既对行业趋势有敏锐判断，又能接触到顶尖的构建者。"梅恩回忆决策过程时说道。在与机构及家族办公室接洽并完成首轮2000万美元募资后，合伙人将首期基金目标设定为1亿美元，目前已开始投资。

零样本基金已投资OpenAI早期产品经理安吉拉·姜（Angela Jiang）的初创公司Worktrace AI。该公司正开发基于AI的管理软件平台，通过智能识别可自动化环节帮助企业实现任务自动化。据PitchBook估计，Worktrace AI已从米拉·穆拉蒂（Mira Murati）及OpenAI基金等知名投资者处获得1000万美元种子轮融资。团队还投资了专注新一代AI增强工厂机器人的Foundry Robotics，该公司近期刚获得由科斯拉领投的1350万美元种子轮。零样本投资的第三家初创公司仍处于保密阶段。

基金创始人表示，他们比多数风投更懂AI发展方向，这既有助于筛选项目，也能规避某些领域。例如梅恩不看好多数情绪编程迭代，他认为拥有编码专长的模型制造商将迅速让此类平台的订阅服务失去价值。森川则凭借对AI与机器人的深刻理解，对当前众多"以人体工学为中心的视频数据机器人公司"持保留态度，这类初创专注于机器人具身训练数据。"研究界正寄望有人能破解具身化迁移难题，"森川评价此类视频数据公司，"但这远未实现。"

梅恩对多数"数字孪生"初创同样持怀疑态度，他在尽调过程中甚至构建推理模型进行测试，结论是普通大语言模型已足够胜任。"预判模型演进方向需要真本事，因为这绝非线性发展。"森川强调。

除创始投资人外，零样本基金还邀请到多位知名顾问，他们将分享基金收益的附带权益。顾问团队包括OpenAI前人力资源主管戴安·尹（Diane Yoon）、OpenAI与苹果前传播主管史蒂夫·道林（Steve Dowling），以及OpenAI前产品负责人卢克·米勒（Luke Miller）。

### 2. Artemis II宇航员能看到哪些卫星未曾捕捉到的景象？
*NPR Topics: News* | 20:11

美国国家航空航天局（NASA）的"阿尔忒弥斯2号"任务四人组于当地时间周一（11月25日）看到了人类从未见过的月球部分。NASA的"猎户座"太空舱上周搭载太空发射系统火箭从肯尼迪航天中心发射升空，这是50多年来首次将人类送上探月任务。

周一的绕月飞行标志着人类有史以来最远的太空旅行。周一（11月25日）下午1时57分，宇航员们超越了"阿波罗13号"任务24.8655万英里（约40万公里）的距离纪录。据NASA称，这次绕月飞行将持续约7小时，期间机组人员将透过舷窗进行实时观测。

宇航员还将获得从相反视角观测日食的独特机会，目睹太阳消失在月球背后。尽管NASA确实拥有来自月球勘测轨道飞行器等卫星的月球照片，但该机构的一位行星科学家在上周末的新闻发布会上解释了为什么人眼和人类观测对理解月球至关重要。

"阿尔忒弥斯2号"月球科学负责人凯尔西·杨周日对记者说："我知道我们将获得的数据会激励下一代科学家和探险家，但它也会让月球离我们更近，让我们所有人团结起来。"杨当天佩戴着描绘日食图案的耳饰。

周一早上，三名NASA宇航员里德·怀斯曼、维克托·格洛弗和克里斯蒂娜·科赫，以及加拿大航天局宇航员杰里米·汉森在距离月球仅1.883万英里（约3万公里）的地方醒来，耳边回响着曼迪莎和托比·麦克的歌曲《早安》。

在绕月飞行期间，机组人员将获得月球作为完整圆盘的独特视角。杨说，在任何特定时刻，月球都有一半被照亮，但在任务最接近月球时，月球背面将有约21%被照亮。

"阿尔忒弥斯2号"为这次绕月飞行设定了10个科学目标，其中之一是观测月球表面的颜色变化。杨说："人眼，尤其是当它与训练有素的大脑相连时——我向你们保证这四个人都具备——能够在眨眼之间做出细致的颜色观察。"

杨说，宇航员将在绕月飞行期间指出"细微的颜色差别"，尤其是在前所未有地看到月球背面时。杨用沙箱作类比：在海滩上，沙子并不完全平坦，它有纹理，颗粒大小也不同。杨描述了观察沙箱的两个角度。

将光线直接照射在沙箱顶部，"你会看到表面的颜色深浅和反照率"。但使用同一个手电筒并将其移到沙箱侧面，"你会失去所有的颜色差别，但你会看到地形和形态"。将这一类比应用于绕月飞行，虽然太阳——光源——不会移动，但航天器会移动。因此，杨说，宇航员将能够从不同的照明角度多次观察同一地点。

杨说："我们知道月球是由什么构成的，我们了解地形，但我们不知道从科学角度来看，机组人员在这些特定的照明条件下会看到什么。这令人兴奋。"机组人员将飞越之前的两处人类着陆点——"阿波罗12号"和"阿波罗14号"，并瞥见月球南极的一小部分，NASA称人类最早可能在2028年登陆那里。

MCP Eats 72% of Your Context Window at Startup
A standard MCP setup can consume around 72% of your available context window before the agent takes a single action. Connect three servers — GitHub, Playwright, and an IDE integration — on a 200K-token model, and tool definitions alone occupy roughly 143K tokens. The agent hasn’t done anything yet. It’s already three-quarters full.
The cost isn’t just tokens. The more unrelated content packed into context, the weaker the model’s focus on what actually matters. A hundred tool schemas sitting in context means the agent wades through all of them on every decision. Researchers have documented what they call context rot — degraded reasoning quality from context overload. In measured tests, tool selection accuracy dropped from 43% to below 14% as tool count increased. More tools, paradoxically, means worse tool use.
The root cause is architectural. MCP loads all tool descriptions in full at session start, regardless of whether the current conversation will ever use them. That’s a protocol-level design choice, not a bug — but the cost scales with every tool you add.
Agent skills take a different approach: progressive disclosure. At session start, an agent reads only each Skill’s metadata — name, one-line description, trigger condition. A few dozen tokens total. The full Skill content loads only when the agent determines it’s relevant. Think of it this way: MCP lines up every tool at the door and makes you choose; Skills gives you an index first, full content on demand.
CLI tools offer a similar advantage. An agent runs git --help or docker --help to discover capabilities on demand, without preloading every parameter definition. Context cost is pay-as-you-go, not upfront.
At a small scale, the difference is negligible. At production scale, it’s the difference between an agent that works and one that drowns in its own tool definitions.
MCP’s Passive Architecture Limits Agent Workflows
MCP is a tool-calling protocol: how to discover tools, call them, and receive results. Clean design for simple use cases. But that cleanness is also a constraint.
Flat Tool Space with No Hierarchy
An MCP tool is a flat function signature. No subcommands, no awareness of session lifecycle, no sense of where the agent is in a multi-step workflow. It waits to be called. That’s all it does.
A CLI works differently. git commit, git push, and git log are completely different execution paths sharing a single interface. An agent runs --help, explores the available surface incrementally, and expands only what it needs — without front-loading all the parameter documentation into context.
