## Structure

Each skill follows the standard Kimi `SKILL.md` convention:

```
skill-name/
├── SKILL.md
├── research/          # (optional) source material & distilled findings
├── templates/         # (optional) JSON templates, prompts, etc.
└── scripts/           # (optional) supporting automation scripts
```

## Honesty boundary

These are **framework distillations** or **personal workflow automations**, not universal products. Character-based skills include explicit capability boundaries and information cut-off dates.
