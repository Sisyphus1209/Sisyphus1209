# PDF 图纸差异对比 · 智能审图

## 使用说明

本 skill 包含三个互补工具，覆盖从原始视觉对比到 AI 结论自动落图的全链路：

| 工具 | 命令 | 用途 |
|------|------|------|
| **视觉对比** | `python scripts/pdf_compare.py "公司图.pdf" "客户图.pdf"` | 像素级逐页对比，输出左右红框对照图 + Markdown 文字 diff 报告 |
| **智能对比** | `python scripts/pdf_compare_smart.py "公司图.pdf" "客户图.pdf"` | 按工程语义分区（标题栏、参数表、技术要求、尺寸标注）进行对比，输出带批注的客户图 + 结构化报告 |
| **AI 批注落图** | `python scripts/pdf_annotate_from_analysis.py "客户图.pdf" "analysis.txt" --text-mode` | 将 AI 审图结论中的差异点自动定位并绘制到客户图纸上 |

## 前置依赖

```bash
pip install pymupdf pillow opencv-python numpy
```

## 一、视觉对比模式（pdf_compare.py）

### 原理
1. 将两本 PDF 逐页转为 200 DPI 高清图
2. 统一尺寸后做灰度差分
3. 形态学处理（闭运算连块、开运算去噪）
4. 找出差异轮廓并用红框标注
5. 同时提取文字做 `unified_diff` 对比

### 输出
- `output/pdf_compare/page_XX_diff.jpg` — 左右红框对照图（中间 10px 白缝）
- `output/pdf_compare/diff_report.md` — 逐页文字差异报告

### 适用场景
- 快速初筛：两版图纸改了哪里
- 客户返图是否有未告知的变更

## 二、智能对比模式（pdf_compare_smart.py）

### 原理
1. 提取 PDF 文字块及其精确 bbox 坐标
2. 按位置和语义将文字分类到 5 个工程区域：
   - **标题栏**：右下角 + 含"设计/校对/审核/比例/版本"等关键词
   - **参数表/性能表**：含减速比、转矩、转速、精度、Nm、r/min 等
   - **技术要求/注释**：含"注："、"技术要求"
   - **尺寸标注**：含 φ、Φ、M、±、°、MAX、MIN 等机械符号
   - **其他文字**：剩余内容
3. 分区域做 diff，并在客户图上用红框+箭头+批注标注差异

### 输出
- `output/pdf_compare_smart/annotated_customer_drawing.jpg` — 带批注的客户图
- `output/pdf_compare_smart/smart_report.md` — 结构化区域差异报告

### 适用场景
- 工程审图：快速定位"参数变了没有"、"标题栏版本是否一致"、"技术要求有无新增"

## 三、AI 批注落图模式（pdf_annotate_from_analysis.py）

### 原理
1. 读取 AI 审图分析结果（支持结构化 JSON 或纯文本）
2. 根据 `target_keywords` 在客户图 PDF 中搜索定位
3. 对匹配位置聚类、合并包围框
4. 在图纸上绘制：红框 + 箭头 + 批注文字 + 严重度标签

### 输入格式

#### 模式 A：JSON 结构化
```json
[
  {
    "target_keyword": "14-φ11通",
    "location_hint": "左上角法兰视图",
    "difference": "安装孔形式不同：客户图为通孔，公司标准图为盲孔",
    "severity": "重大差异"
  }
]
```

#### 模式 B：纯文本（直接复制 Kimi/AI 回答）
```
一、安装法兰接口（重大差异）
结论：虽然分度圆均为φ233±0.1，但因安装孔形式完全不同...

二、输入端轴向尺寸（关键差异）
风险点：改制版输入端长出5mm，可能导致与联轴器/电机的轴向干涉...
```

### 输出
- `output/pdf_annotated/annotated_drawing.jpg` — 自动标注后的图纸
- `output/pdf_annotated/annotation_summary.md` — 标注点汇总

### 适用场景
- 已有 AI 审图结论，需要把文字结论转化为可视化图纸批注发给客户/车间

## 核心设计决策

- **DPI 锁定 200**：工程图纸的甜点，足够看清文字和线条，又不会让 CV 处理过慢
- **差分阈值 30**：白底黑线的图纸中，30 灰度差足够敏感，又不被扫描噪点干扰
- **轮廓过滤 25×25**：过滤掉微小的字体渲染差异，只标出有意义的区域
- **关键词降级搜索**：先精确匹配，再模糊匹配（去掉 φ、±、空格等干扰字符）

## 诚实边界

- 本工具依赖 PDF 中的**可选文字层**。如果图纸是扫描件（纯图片无 OCR），则文字提取和关键词定位会失效，视觉对比模式仍可用
- 批注落图模式中的关键词猜测基于 RV 减速器/机械制图经验，对非机械类文档可能需要手动调整 `guess_keywords`
- 中文批注依赖系统中存在 `msyh.ttc`、`simhei.ttf` 或 `simsun.ttc` 字体，否则回退到默认字体
