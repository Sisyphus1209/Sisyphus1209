# 微博 V+ 问答抓取器

## 使用说明

本工具用于抓取指定微博账号的 V+ 付费问答内容，采用**手动控制模式**（用户要求），不会自动完成付款或跳转。

## 快速开始

### 方式 1：运行最新版脚本

```bash
cd weibo-crawler
python erzong_vplus_scraper_v5.py
```

或双击 `抓取二总V+问答v4.bat`

### 方式 2：启动脚本菜单

双击 `启动.cmd` 或运行：

```bash
python 启动脚本.py
```

## 手动控制流程

1. 脚本自动打开浏览器并导航到目标 V+ 问答页面
2. **停在页面等待用户操作**（不会自动点击"购买"或"查看答案"）
3. 用户手动完成必要的登录/付费/展开答案
4. 用户回到终端按提示确认继续
5. 脚本开始抓取当前可见的问答内容

## 输出

- `output/` 目录下生成抓取的问答内容（Markdown / 文本格式）
- `links.txt` / `links_backup.txt` 记录已抓取的链接，用于断点续传和去重

## 主要脚本

| 脚本 | 说明 |
|------|------|
| `erzong_vplus_scraper_v5.py` | 最新稳定版，手动控制模式 |
| `erzong_vplus_scraper_v4.py` | v4 版本 |
| `weibo_vplus_scraper_v3.py` | 早期版本 |
| `debug_page.py` | 页面结构调试 |
| `debug_qa_page.py` | 问答详情页调试 |

## 重置与维护

- `重置登录状态.bat`：清除本地登录缓存，重新扫码登录
- `debug_article_tab.py`：调试文章 Tab 页结构

## 诚实边界

- **必须遵守微博平台规则**，本工具仅供个人学习研究使用
- **手动控制**是为了避免违反用户"不要自动跳转付款页面"的要求
- 抓取内容受版权保护，请勿公开传播或商用
- 微博页面结构变化可能导致脚本失效，需要配合调试脚本手动修复选择器
