# 微博 V+ 问答抓取工具 (优化版)

## 主要改进

### 1. 登录状态持久化
- 使用 `storage_state` 保存完整的浏览器状态（包括 cookies 和 localStorage）
- **第一次扫码后，登录状态会保存，后续运行不需要再扫码！**
- 登录状态文件：`output/weibo_storage_state.json`

### 2. 自动取消分享勾选
- 点击"免费围观"按钮前，会自动取消"分享到微博"的勾选
- 避免自动发微博的困扰

### 3. 优化的内容提取
- 针对 V+ 问答页面结构优化
- 分离问题和答案内容
- 自动提取发布时间

## 使用方法

### 第一次运行
1. 双击运行 `抓取二总问答.bat`
2. 等待浏览器打开并显示二维码
3. 使用微博手机 APP 扫码登录
4. 登录成功后脚本会自动保存登录状态
5. 等待抓取完成

### 后续运行
- 直接双击运行 `抓取二总问答.bat`
- **无需再次扫码！** 脚本会自动加载之前保存的登录状态

### 如果登录过期
- 删除 `output/weibo_storage_state.json` 文件
- 重新运行脚本，扫码登录即可

## 输出文件

抓取结果会保存在 `output/` 目录：
- `erzong_qa_YYYYMMDD_HHMMSS.json` - JSON格式数据
- `erzong_qa_YYYYMMDD_HHMMSS.md` - Markdown格式（推荐阅读）
- `erzong_qa_YYYYMMDD_HHMMSS.txt` - 纯文本格式

## 注意事项

1. **登录状态有效期**：微博登录状态通常可以保持几天到几周，取决于微博的策略
2. **抓取频率**：脚本内置了随机延迟，避免被微博限制
3. **V+内容**：对于需要"免费围观"的内容，脚本会自动处理

## 常见问题

**Q: 为什么第一次还要扫码？**
A: 第一次需要建立登录状态。扫码后状态会自动保存，以后就不需要了。

**Q: 登录状态能维持多久？**
A: 通常几天到几周不等。如果发现需要重新登录，说明状态已过期，重新扫码即可。

**Q: 抓取的内容还是空的？**
A: 检查 `output/debug_qa_*.png` 截图，看看页面是否正确加载。如有问题请反馈。
